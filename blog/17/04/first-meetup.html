My first software meetup

So this evening I attended my first software developers meetup in my local town - Taunton.
The talk was titled 'An Introduction to Xamarin Forms' and was presented by Martin Rock-Evans a lead software developer at UKHO.

A couple weeks back I saw the new Github advert which made me think about attending a hackathon. So I looked around online to find some local hackathons but all I found was yearly events that I had already missed, or groups in far, far away London town. The closest thing I could find was the Taunton (and area) Developers Meetup on meetup.com. Luckily, the next event was only in a couple of weeks so I joined the group, checked if it all okay for me to attend and so I went along.

Me and my friend Will caught the train over to Taunton, grabbed a coffee and made our way over to the venue. Martin started by giving us a run through of the basic capabilities of Xamarin and the idea behind it (one single C# shared code-base) and some of the features it had. He quickly made up a small demo app with a ListView and some other small components. What I was comparing Xamarin to was Fuse. Fuse is a cross-platform app development tool that I have been using on and off for about a year now, and it's excellent.

There were lots of similarities between Xamarin and Fuse that I didn't expect at first, almost all of the UI layout and even different class names and the syntax for MVVM (Model View View Model) components was very similar. They both use their own take on an XML based UI layout, Fuse's being called UX and Xamarin's called XAML. Their MVVM components were both called Observables (pretty much) and to bind to back end code both used similar syntax Value="{variable}" for Fuse and Value="{Binding variable}" for Xamarin. The biggest thing I seemed to notice was the amount of excess code needed for the base of an application in Xamarin. Using C# as the whole backend obviously means you must have you full Main class to begin with where as in Fuse you can just start with a UX file only and have a running app. What's more, Fuse doesn't depend on the whole .NET backend. When being installed on a device Xamarin installs the .NET binary alongside the code base binary that you have written. When in Fuse (as far as I know), their in-house compiler accesses the native APIs directly - although don't quote me on this as I remember Martin saying something about direct API access with Xamarin but I can't quite remember if that was through the .NET binary or straight to the device APIs. In general I like Fuse because of their progressive startup nature (just my opinion) and the way that they allow their customers much easier app development all-round, even more so than Xamarin I think.

On the contrary there are a few things that may side you with Xamarin: the fact that it is now owned by Microsoft obviously means that the development will be very very fast in comparison with Fuse and will have a lot more native SDKs available for you to use. However it's not too hard for you to use foreign code SDKs in Fuse at all, many members of the community have implemented SDKs such as Firebase, SQLite and others. So make your own decision.

At the end of it all, I really enjoyed being around people with the same interest as me. It's not often I get to chat with other programmers so it was really refreshing and hopefully I've made some connections that will help me in the future! So thank you very much Martin for the talk, Jonn and Ewen for organising the Meetup group and everyone else who came along to make it possible - I look forward to the next one!
