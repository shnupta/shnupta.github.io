bric

bric is an in-terminal text editor (like nano) based of off kilo by antirez. It is my first open-source project that I have had to manage contributions by others from across the world.

In April of 2017 I came across a blog post by antirez (Salvatore Sanfilippo) on how he had created an editor in less than 1000 lines of code. He called this kilo. I read through his post and decided that I would write my own clone of the editor and thus bric was created.

I can't quite remember how long I spent writing my clone but it was probably around a week considering I would have been at school each day. I learnt a lot of things by making a clone of this editor, mostly to do with the terminal but it also strengthened my knowledge of memory management and pointers etc. Once I had finished cloning the editor I decided that I'd add some more features to it as it was super basic how antirez had left it (which is what he intended). The first change I made was to implement a modules function that would be the base for future changes to the editor. Although it is very basic it has so far served its purpose. I then went ahead and started to implement some of the Vim functions that I use regularly, such as goto, find and replace, skip to end of line and others.

After adding these I began to think, why not change bric to a modal editor (like Vim) as this will allow for much superior editing that just a basic single mode editor like nano. With the help of a couple new contributors I implemented three new modes: normal, insert and selection mode.
