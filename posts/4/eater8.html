Eater8

The other day I was browsing on reddit (as you do) and came across a post on r/compsci that was asking what are people's out of work hobbies. Someone commented that they were following Ben Eater's videos on building an 8-bit breadboard computer.

I visited his channel and watched a few videos until I stumbled across one where he programs the computer. This got me thinking...
I should write a little emulator of Ben's computer and see if I can create some interesting programs on it.

So in about 30 minutes I modelled the computer's basic architecture (RAM, registers, program counter, etc.). Ben's computer had two registers - A and B. Each of those hold 8 bits so I modelled them with uint8_t A; and the same for register B.

I wasn't sure of the exact size of RAM (I will get in touch with Ben sometime to get the correct details) but I just left it as 512 bytes of RAM by using a uint8_t RAM[512];

Finally I would need a program counter to point to the memory adresses in RAM and also a current opcode to decode - both of which I just modelled similarly to the A and B registers. I also used a bool halt; that would be set to true when the halt opcode is called, as well as an int out; for the output of the program.

In Ben's video he briefly shows some of opcodes he has made for the computer. These opcodes are in binary which makes them slightly harder to read but nonetheless there were only about 3 opcodes actually needed for this program he was creating.

What he programmed was a simple addition program that loaded a number into the A register (opcode LDA - 0001xxxx where xxxx is the memory address of the value to load into A), then add a number to the value in the a register (opcode ADD - 0010xxxx where xxxx is the memory address of the value to add to the A register) and then finally output the value in the A register (opcode OUT - 01010000).

All that I have emulated so far is this exact program that Ben made in the video but when I get time I will contact Ben, model the computer correctly and also complete the instruction set. And then I will allow the user to load binary files into the emulator that will then be able to run programs designed specifically for it.
